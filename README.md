## Быстрый старт.

1) Клонировать репозиторий.

2) Скомпилировать проект
```shell
cmake -B build && cmake --build build
```

3) Запустить проект
```shell
./build/dd_calc -x 124.24312412434234 -c
```

### О режимах работы программы, получаемые флаги.

У программы есть несколько режимов работы.
1) Просто посчитать значение функции в какой-то заданной точке —  `-x [num]`
2) Посчитать значение в точке и сравнить его с результатом функции стандартной библиотеки — `-x [num] -c`
3) Посчитать значение в точке и провести замеры по времени на n итерациях — `-x [num] -b -n [count]`


## Запуск графиков

Пока есть только один график — сравнение быстродействия реализованной функции с функцией стандартной библиотеки. Точку, в которой строится график, можно задать самостоятельно. График относительной погрешности не добавлен, так как она является константой для фиксированного аргумента.
```shell
./benchmark.sh [num]
```
## Сопроводительная записка

### Реализация низкоуровневых операций

Для корректной работы арифметики двойной точности были реализованы базовые операции над числами с плавающей точкой: `quick_two_sum`, `two_sum`, `two_diff`, `two_prod` и `split`. 

### Структура double-double

На основе указанных операций реализована структура `dd`, представляющая число в виде пары `(hi, lo)`, где `hi` — старшая часть, `lo` — поправочный член, хранящий потерянные биты. Для структуры определены операторы `+`, `-`, `*`, `+=`, а также вспомогательные функции `mul`, `square`, `ldexp`.

### Функция exp

Реализована функция `exp : dd → dd`. В основе алгоритма лежит преобразование аргумента и вычисление ряда Тейлора.

**Основная идея.** Аргумент представляется в виде `a = m·ln(2) + k·r`, откуда `exp(a) = 2^m · exp(r)^k`. Малость `r` обеспечивает быструю сходимость ряда Тейлора, а множитель `2^m` восстанавливается через `ldexp`.

**Детали.** Константа редукции выбрана равной `k = 512 = 2^9`. Такой выбор обусловлен балансом между двумя противоположными эффектами: при меньшем `k` ряд Тейлора сходится медленнее, при большем - накапливается погрешность из за слишком малого `r`. Значение `k = 512` обеспечивает сходимость за ~15 итераций. Поскольку `k` является степенью двойки, возведение `exp(r)^k` выполняется за `log2(512) = 9` шагов быстрым возведением в степень.

### Интерфейс командной строки

С помощью библиотеки CLI11 добавлен CLI со следующими параметрами:

| Флаг | Описание |
|------|----------|
| `-x`, `--value` | Аргумент функции (обязательный) |
| `-c`, `--compare` | Сравнение с `std::exp` и вывод относительной погрешности |
| `-b`, `--benchmark` | Замер времени выполнения |
| `-n`, `--iterations` | Число итераций для замера (по умолчанию 1 000 000) |
